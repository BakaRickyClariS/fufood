# 團隊反饋統整 (2025-12-31)

本文件整理自 2025/12/31 的團隊對話紀錄，主要涵蓋掃描、入庫、清單及登入流程的問題與建議。

## 🐛 錯誤回報 (Bugs)

### 1. 清單功能 (List Functionality)

- **一次新增多個項目失敗**
  - **描述**: 在清單中嘗試一次增加兩個項目時會發生錯誤/失敗。
  - **原始反饋**: "我一次增加兩個清單項目會失敗欸"

### 3. 登入與其他 (Authentication & Others)

- **LINE QRCode 登入未導回 (Redirect Issue)**
  - **描述**: 掃描 QRCode 並在瀏覽器登入 LINE 後，畫面停留在白色的「Success」頁面，未自動導回 App 的上一頁或首頁。
  - **原始反饋**: "點了QRCode 會跳到瀏覽器 然後登入line 然後就變這樣", "應該送出後要導回上一頁"

- **群組數量限制失效 (Group Limit Issue)**
  - **描述**: 使用者持有的群組數量超過了限制（3個）。
  - **原始反饋**: "群組不是三個而已嗎？我超過三個"

- **QR Code 彈窗無法關閉 (Uncloseable QR Modal)**
  - **描述**: 打開 QR Code 邀請視窗後，無法將其關閉（可能缺少關閉按鈕或點擊無效）。
  - **原始反饋**: "QR碼打開關不回去..."

## 💡 優化建議 (UX Improvements)

### 1. 入庫流程 (Stock In Workflow)

- **缺少刪除按鈕 (Missing Delete)**
  - **描述**: 入庫介面的品項清單中，缺乏刪除單一項目的按鈕。
  - **原始反饋**: "入庫沒有刪除的鈕"

- **入庫後引導 (Post-Action Navigation)**
  - **描述**: 完成入庫後，目前可能缺乏明確指引。建議增加彈跳視窗（Modal/Popup）詢問使用者：
    1. 前往查看庫存
    2. 繼續進行入庫
  - **原始反饋**: "要多個彈跳通知 詢問前往察看庫存 還是繼續進行入庫", "新增需求送出的時候要多個彈跳通知 詢問前往察看庫存 還是繼續進行入庫"

- **送出按鈕反饋 (Submission Feedback)**
  - **描述**: 點擊「確認入庫」後缺乏狀態鎖定與反饋，容易導致重複提交。
  - **建議解法**: 點擊送出後，將按鈕鎖定並顯示「送出中」狀態。送出完成後，維持鎖定狀態避免重複點擊，並跳出成功彈窗（Success Popup）進行後續引導。
  - **原始反饋**: "如果直接去確認入庫 應該要通知有確認 不然不知道就會看到庫存有5顆飯糰(因為按5次)", "送出按鈕再送出時把它鎖定變成送出中好了 送完再跑出彈跳視窗並顯示送出完成鎖定住 避免重複入庫"

### 2. 登入流程 (Login Flow)

- **跳過開始畫面 (Skip Start Screen)**
  - **描述**: LINE 登入成功後，目前會先導向開始畫面再進入首頁。使用者希望可以直接跳過開始畫面，直接進入首頁。
  - **原始反饋**: "登入加入賴之後 會跳轉到開始畫面 然後進入首頁 有機會跳過開始畫面 直接進入首頁嗎？"
